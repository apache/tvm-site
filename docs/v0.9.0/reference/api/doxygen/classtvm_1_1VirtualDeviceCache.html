Detailed Description

A cache of VirtualDevices. This can be used:

  • To avoid ending up with lots of identical instances, since the space of VirtualDevices for any one compilation is very small but the number of points they need to be constructed can be very large (eg during device planning).
  • So we can assume VirtualDevices are pointer equal if and only if they are structurally equal. This simplifies the unification of 'device domains' which are built on VirtualDevices.
