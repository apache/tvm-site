1149 /* \brief Allow relax.Var as key in STL tables
1150  *
1151  * For most Relax expressions, it would be ambiguous whether the
1152  * expression should follow reference equality or structural equality.
1153  * This is not the case for variables, which do not contain nested
1154  * internal structure, and are frequently used as keys in lookup
1155  * tables.
1156  *
1157  * Providing `std::hash` and `std::equal_to` specializations for
1158  * `relax::Var` allows it to be used as a key in STL tables. For
1159  * `relax::Expr`, the user must specify the type of equality used
